# ESU-Admin 项目介绍

## 一、项目概述

**ESU-Admin** 是一个酒店管理后台系统，面向携程 Web 夏令营 2026（ctrip-web-camp-2026）。项目处于 v1.0 阶段，基础业务逻辑已实现，界面和部分数据流仍在完善中。

## 二、技术栈

| 类别     | 技术                         |
|----------|------------------------------|
| 前端框架 | **Preact**（类 React 的轻量框架） |
| 路由     | **preact-iso**               |
| 状态管理 | **@preact/signals**          |
| 构建工具 | **Vite 7**                   |
| 语言     | **TypeScript**               |

## 三、核心功能

### 1. 用户角色

- **商家（merchant）**：管理自己录入的酒店
- **管理员（admin）**：审核、管理所有酒店

### 2. 商家端

- **我的酒店**：查看名下酒店列表及审核状态
- **新增录入**：录入新酒店信息
- **酒店编辑**：修改酒店信息（名称、地址、星级、设施、房型等）
- **删除酒店**：删除已录入的酒店

### 3. 管理端

- **审核中心**：查看待审核、已通过、已拒绝、已下线的酒店
- **审核操作**：通过、拒绝（含拒绝原因）、下线、重新上线
- **酒店详情**：查看完整酒店信息及房型

### 4. 酒店状态

- 审核中 → 通过 / 不通过
- 通过 → 已下线
- 已下线 → 重新上线

## 四、项目结构

```
src/
├── api/           # 接口封装
│   ├── request.ts # 请求封装、Token 管理
│   ├── auth.ts    # 登录、注册、获取当前用户
│   ├── hotel.ts   # 酒店 CRUD、审核接口
│   └── roomType.ts# 房型接口
├── pages/         # 页面
│   ├── Login.tsx      # 登录
│   ├── MerchantList.tsx # 商家酒店列表
│   ├── HotelEdit.tsx   # 酒店编辑/新增
│   └── AuditList.tsx   # 管理员审核中心
├── components/    # 公共组件
│   ├── Toast.tsx      # 提示
│   └── ConfirmDialog.tsx # 确认弹窗
├── store.tsx      # 全局状态（用户、酒店列表、Toast、确认框）
├── config.ts      # API 基础地址配置
└── app.tsx        # 应用入口与路由
```

## 五、后端对接

- API 根地址通过环境变量 `VITE_API_BASE` 配置
- 使用 Token 鉴权，请求头中携带 `Authorization: Bearer <token>`
- 主要接口：`/users/login`、`/users/me`、`/hotels/*` 等
